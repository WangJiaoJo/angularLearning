NgModule

  • NgModule Introduction
    • An application has at least one module, which is the root module.
    • configure the injector and the compiler
      • import other modules
      • declare components, directives and pipes belong to this module
      • make components, directives and pipes be public
      • provide services to the application
    • organize related things together
  • Declarations
    • A list of declarable classes, (components, directives, and pipes) that belong to this module.
    • Components, directives, and pipes must belong to exactly one module.
  • Providers
    • Dependency Injection about how to obtain value for a dependency. The dependencies are the services created.
    • In user.service.ts, provideIn: 'root' specifies that the service should be provided in the root injector, this service will be available throughout the app.
    • In user.service.ts, provideIn: UserModule specifies that the service should be provide in the particular NgModule, this service will be available unless the app import the UserModule.
    • In user.module.ts, providers: [UserService] declares a provider for the service within the module.
    • In app.component.ts, providers: [UserService] providing a service in the component limits the service only to that component, other components in the same module can't access it.
  • Imports
    • A list of module which should be folded into this module. Folded means it is as if all the imported NgModule's exported proporties were declared here.
    • Specifically, it is as if the list of modules whose exported components, directives, or pipes are referenced by the component templates were declared in this module.
    • A component template can reference another component, directive, or pipe when the reference is declared in this module or if the imported module has exported it.
  • Exports
    • Importing a module does not automatically re-export the imported module's imports.
    • Module B imports Module A, Module A imports the commonModule to use NgIf and does not export the commonModule, Module B cannot use NgIf. If Module A re-exports the commonModule, Module B can use NgIf.
  • Bootstrap
    • A list of components that are automatically bootstrapped.
    • Usually there's only one component in this list, the root component of the application.
    • Angular can launch with multiple bootstrap components, each with its own location in the host web page.
    • A bootstrap component is automatically added to entryComponents.
  • entryComponents
    • A list of components that can be dynamically loaded into the view.
    • By default, an Angular app always has at least one entry components, the root component, AppComponent. Its purpose is to serve as a point of entry into the app, that is, you bootstrap it to launch the app.
    • Routed component are also entry components because they need to be loaded automatically.
    • While the bootstrapped components are entry components, you don't have to add them to a module's entryComponents list, as they are loaded implicitly.
    • Angular automaticall adds components in the module's bootstrap and route definition into the entryComponents list.
    • Dynamic component loading is not common in most apps beyond the router. If you need to dynamically load components, you must add these components to the entryComponents list yourself.
