NgModule

  • NgModule Introduction
    • An application has at least one module, which is the root module.
    • configure the injector and the compiler
      • import other modules
      • declare components, directives and pipes belong to this module
      • make components, directives and pipes be public
      • provide services to the application
    • organize related things together
  • Providers
    • Dependency Injection about how to obtain value for a dependency. The dependencies are the services created.
    • In user.service.ts, provideIn: 'root' specifies that the service should be provided in the root injector, this service will be available throughout the app.
    • In user.service.ts, provideIn: UserModule specifies that the service should be provide in the particular NgModule, this service will be available unless the app import the UserModule.
    • In user.module.ts, providers: [UserService] declares a provider for the service within the module.
    • In app.component.ts, providers: [UserService] providing a service in the component limits the service only to that component, other components in the same module can't access it.
